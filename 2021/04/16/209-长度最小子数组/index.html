长度最小子数组

题目

给定一个含有 n 个正整数的数组和一个正整数 target 。

找出该数组中满足其和 ≥ target 的长度最小的 连续子数组 [numsl, numsl+1, …, numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。

示例一：

输入：target = 7, nums = [2,3,1,2,4,3]

输出：2

解释：子数组 [4,3] 是该条件下的长度最小的子数组。

示例 2：

输入：target = 4, nums = [1,4,4]

输出：1

示例 3：

输入：target = 11, nums = [1,1,1,1,1,1,1,1]

输出：0

提示：

  • 1 <= target <= 109
  • 1 <= nums.length <= 105
  • 1 <= nums[i] <= 105

进阶：

如果你已经实现 O(n) 时间复杂度的解法, 请尝试设计一个 O(n log(n)) 时间复杂度的解法。

我的写法（滑动窗口法）

  滑动窗口法有点类似于快慢指针法，都是一前一后两个指针。本题要求找到一个数组中其和大于某个数的最小子数组，滑动窗口法顾名思义就是滑动窗口寻找符合条件的情况。那么我们可以设置left和right两个指针代表窗口的左右边界。如果这个区间内所有值的和大于target，那么就算成功找到一个子区间，这时要通过增加左边界来缩小区间。幸运的话缩小后的区间也是一个合格子区间，那么这个子区间会是一个更优解。如果区间内所有数的和小于target的话，毫无疑问要通过增大右边界来增大区间了。
